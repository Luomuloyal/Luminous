二、作品简介
Luminous是一款面向大众用户的智能健康管理App，聚焦于用药安全与健康习惯养成。核心功能包括：
药品拍照识别：通过AI图像识别技术，用户可拍照快速识别药品信息。
药品信息查询与AI解读：集成权威药品数据库，结合AI辅助解读药品说明书、用法用量及注意事项。
智能用药提醒与打卡：支持自定义提醒、周期性打卡，帮助用户规范用药。
健康数据管理：本地与云端结合，保障数据安全与隐私。
多端适配：支持iOS、Android，后端可独立部署。
本项目适合个人健康管理、家庭成员用药监督、慢病患者长期用药等多场景。


创新性描述：本项目创新性地融合AI图像识别与智能问答技术，实现药品信息一拍即得、用药方案智能推荐，并通过多端同步和本地隐私保护，提升用户健康管理的智能化和便捷性。

三、开源代码与组件使用情况说明
1. 自主开发部分：
   App主业务逻辑、UI界面、健康数据管理、用药提醒等均为团队原创开发。
2. 开源组件集成：

   智能问答：对接豆包/火山方舟等AI平台，实现药品相关问题的智能解答。
4. 后端开源依赖：
   Node.js、TypeScript、Express、MongoDBMySQL等。
5. 依赖详情：详见pubspec.yaml与backend/package.json。

四、作品安装说明
1. 环境准备
   安装Flutter SDK（建议3.11.1及以上）
   安装Dart环境
   配置Android Studio/Xcode等开发工具（用于真机/模拟器调试）
   安装Node.js（后端服务需要）
   安装MongoDB、MySQL（可选，或使用Docker一键部署）
2. 获取代码
   克隆本项目仓库：
   git clone <项目仓库地址>
3. 安装依赖
   进入项目根目录，执行：
   flutter pub get
   后端依赖安装：
   cd backend
   npm install
4. 启动服务
   启动后端服务：
   cd backend
   npm run dev
   启动Flutter App：
   flutter run
5. Docker一键部署（可选）
   项目根目录执行：
   docker compose up -d --build
   默认启动backend、mongodb、redis、mysql等服务。
6. 其他
   详细配置与常见问题见 README.md、../lib-docs/deployment-config.md。

五、设计思路
1. 以用户为中心：聚焦用药安全、健康习惯养成，提供智能化、个性化的健康管理体验。
2. AI赋能：利用AI图像识别、智能问答等技术，降低、用户用药门槛，提升信息获取效率。
3. 前后端分离架构：Flutter负责前端多端适配，Node.js后端负责数据处理与AI服务对接，便于扩展与维护。
4. 数据安全与隐私保护：本地与云端结合，敏感数据加密存储，严格权限管理。
5. 高可用与易用性：支持离线使用、断网数据缓存，界面简洁友好，交互流畅。
6. 可扩展性：架构设计支持后续接入更多健康管理功能，如运动、饮食、慢病管理等。

六、设计重点与难点
1. AI识别准确性：药品包装、说明书等图片种类繁多，需不断优化AI模型与识别流程，提升识别率。
2. 用药提醒实用性：需支持多种提醒周期、复杂用药场景，兼顾灵活性与易用性。
3. 跨平台适配：Flutter需兼容不同设备分辨率、系统特性，保证UI一致性与性能。
4. 数据同步与隐私保护：本地与云端数据同步需防止丢失与冲突，用户健康数据需加密存储，严格权限控制。
5. AI问答的专业性与安全性：需防止AI输出不准确或误导性用药建议，所有AI解读均附权威来源与免责声明。
6. 后端高并发与可扩展性：需支持多用户并发访问，后端服务易于横向扩展。
